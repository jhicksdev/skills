# Project Folder Structure Blueprint

> **Project**: skills — Personal Agent Skills Repository
> **Type**: Agent Skill Collection
> **Last Updated**: 2026-06-26

## 1. Structural Overview

This repository serves as a personal catalog of agent skills — modular packages that extend AI agent capabilities with specialized knowledge, workflows, and tools. The structure is organized around these principles:

- **Skill-centric isolation**: Each skill lives in its own directory under `skills/`, fully self-contained with its own `SKILL.md` and any supporting scripts.
- **Flat skill layout**: All skills sit at the same level under `skills/` for simple discovery. No nested grouping — skills are independent units.
- **Separation of concerns**: Skill source code, evaluation scenarios, and agent configuration are kept in distinct top-level directories.
- **Scaffold-ready**: A `template-skill/` provides a reusable starting point for authoring new skills.

## 3. Key Directory Analysis

### Top-Level Files

| Path | Purpose |
|------|---------|
| `README.md` | Entry point — describes the repo, how to use skills, how to add new ones |
| `PROJECT_STRUCTURE_BLUEPRINT.md` | This blueprint — definitive reference for structure conventions |

### `skills/` — Skill Definitions

Each subdirectory under `skills/` is a self-contained skill. A skill **must** have a `SKILL.md` file and **may** have a `scripts/` directory with executables.

- Skills are named with `kebab-case` (e.g., `code-review-helper`, `react-component-generator`).
- No nesting deeper than one level — every skill is a direct child of `skills/`.
- Shared utilities should be avoided; each skill is meant to be standalone.

### `skills/template-skill/` — Skill Scaffold

A boilerplate skill that serves as a starting point for authoring new skills. Copy this directory and rename it when creating a new skill.

### `eval/` — Evaluation Scenarios

Contains test prompts, expected outputs, and evaluation scripts for measuring skill performance and accuracy. Mirrors the skill structure conceptually but is kept separate from skill source.

### `.opencode/` — Agent Configuration

Configuration files for the opencode AI agent toolchain. Managed by the `opencode` CLI and should not be manually edited unless necessary.

## 4. File Placement Patterns

| File Type | Location | Convention |
|-----------|----------|------------|
| Skill definition | `skills/<skill-name>/SKILL.md` | One `SKILL.md` per skill, no exceptions |
| Skill scripts | `skills/<skill-name>/scripts/` | Executables only (Python, shell, etc.) |
| Skill assets | `skills/<skill-name>/assets/` *(optional)* | Images, templates, reference files |
| Evaluation data | `eval/` | Named to match the skill it tests |
| Agent config | `.opencode/config.json` | Generated by `opencode init` |

## 5. Naming Conventions

- **Skill directories**: `kebab-case` — `code-review-helper`, `api-doc-generator`
- **Configuration files**: `lowercase` — `config.json`, `opencode.json`
- **Documentation**: `UPPER_SNAKE_CASE` — `README.md`, `PROJECT_STRUCTURE_BLUEPRINT.md`
- **Scripts**: `kebab-case` or `snake_case` — `validate.sh`, `run_tests.py`

## 6. Navigation and Development Workflow

### Adding a New Skill

1. Copy `skills/template-skill/` to `skills/<your-skill-name>/`
2. Edit `SKILL.md` with the skill's instructions, triggers, and workflow
3. Add helper scripts to `scripts/` if needed
4. Verify by loading the skill in an AI agent

### Finding a Skill

All skills are in `skills/` — browse the `SKILL.md` files or grep for keywords:

```bash
grep -rl "keyword" skills/*/SKILL.md
```
